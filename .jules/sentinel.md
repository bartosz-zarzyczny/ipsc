## 2024-05-18 - Session Fixation and Secure Cookies\n**Vulnerability:** Session fixation and insecure cookie settings.\n**Learning:** The application was not clearing the session upon login/logout, leading to potential session fixation vulnerabilities. It also missed standard session security configurations `SESSION_COOKIE_HTTPONLY` and `SESSION_COOKIE_SAMESITE`.\n**Prevention:** Use `session.clear()` on authentication boundaries and properly set Flask session security headers in `app.config`.
